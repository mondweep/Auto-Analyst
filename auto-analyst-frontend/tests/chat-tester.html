Auto-Analyst Chat Interface Tester

This tool helps test the chat interface in the Auto-Analyst application.

Setup Instructions:

  1. Start the Auto-Analyst backend services (file_server.py and automotive_server.py)
  2. Start the Auto-Analyst frontend (npm run dev)
  3. Open the chat interface in another browser tab (usually http://localhost:3000/chat)
  4. Open your browser's developer console in that tab (F12)
  5. Copy the code from the chatTester.js file, paste it in the console, and press Enter
  6. Run chatTester.start() in the console to start automated testing

Test Cases

Test Case 1: Chat Interface Loads Properly

Description: Verify that the chat interface loads properly with all necessary components.

Expected Result: Chat interface should load without errors, input field visible, welcome message displayed.

Test Case 2: Send a Basic Question and Receive Response

Description: Test asking a basic question about inventory data.

Test Question: "What vehicles are in the inventory?"

Expected Result: Response should mention Toyota Camry, Honda Civic, Ford F-150 and other inventory items.

Test Case 3: Model Provider Settings

Description: Verify the model provider is correctly set to Gemini and works without API key issues.

Test Question: "Based on the inventory and market data, which vehicle would be the best one to promote in a sale to maximize profit?"

Expected Result: Response with reasoning showing Gemini's capabilities, mentioning profit maximization, price considerations, etc.

Test Case 4: Data Analysis Question

Description: Test the chat interface's ability to analyze data in CSV files.

Test Question: "Analyze the market data and tell me which vehicles are priced below market value"

Expected Result: Response should mention Toyota Camry and Honda Civic being priced below market, with specific percentage differences.
